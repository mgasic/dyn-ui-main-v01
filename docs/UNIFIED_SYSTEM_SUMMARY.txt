KEY PRINCIPLES:
===============

1. TOKEN LAYERS:
   - Foundation tokens are global, immutable
   - Component tokens reference foundation tokens
   - Theme tokens override for dark mode, etc

2. NAMING:
   Format: --dyn-[component]-[property]-[state]
   Examples: --dyn-button-bg, --dyn-button-bg-hover
   
3. FALLBACK:
   Every token must have 3-level fallback
   var(--dyn-component, var(--legacy-component, fallback))

4. NO HARDCODING:
   All values must use tokens (except fallback)

5. DARK MODE:
   Use @media (prefers-color-scheme: dark)
   Override only changed tokens

6. TESTING:
   Min 80% Jest coverage
   Test all variants and states
   Test dark mode and accessibility

7. ACCESSIBILITY:
   Focus ring visible (3px+)
   Color contrast >= 4.5:1
   Touch targets >= 44px
   ARIA labels
